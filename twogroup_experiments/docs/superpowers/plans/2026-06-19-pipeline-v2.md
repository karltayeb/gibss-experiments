# Pipeline v2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Refine the YAML-config pipeline so identity is hashed from config (not code), code-change invalidation is native snakemake, run-code lives in cohesive per-item modules, and reduction/analysis scoping is declarative — without changing the simulate→fit→reduce→analyze behavior.

**Architecture:** Drop the dehydrate/rehydrate serializer: the manifest stores the **config coordinate** (resolved library sub-dicts) and its content hash; rules re-resolve specs from the coordinate via the loader. `SimulationSpec` becomes a transient resolved bundle carrying a loader-set `hash`; `MethodSpec` is removed (fit resolves `fn(**kwargs)` inline). Run-code moves into `simulations/ fits/ reductions/ analyses/` modules (re-exported from `core` so YAML `function:` names resolve). Name-keyed stages (reduce, analyze) become loop-generated per-item `script:` rules (native script-mtime tracking); hash-keyed stages (simulate, fit) stay single generic rules with per-item `code` inputs. `method_filter`/`simulation_filter` become boolean predicates applied at DAG-build.

**Tech Stack:** Python 3.13, polars, numpy, PyYAML, snakemake 9.20, matplotlib, gibss. Tests: `uv run pytest`.

## Global Constraints

- Run all Python via `uv run` (never bare `python`/`pip`).
- Builds on branch `yaml-experiment-config` (v1). Clean break on hashes (nothing to preserve).
- Identity = config: `hash = sha256(canonical_json(coordinate))`. Code bodies MUST NOT enter the hash.
- `inspect.getfile` must point at the real submodule — re-export from `core` with `from <pkg>.<mod> import *` (re-export preserves `getattr(core, name)`; `getfile` follows the true definition).
- Library entry keys never contain `__`. Generated method name = `{base}__{k}={format_over_value(v)}` per `over` key in declared order; float → `format_float` (`.2f`), int bare, bool `true`/`false`, None `none`, str verbatim.
- `defaults`: `base_seed=20260501`, `replicates_per_batch=50`, `n_batches=1`.
- Predicates run at DAG-build on **config** only (no materialized data); cheap + pure.
- Decisions taken (reversible): output-level `method_filter` stays an explicit name-list; `analyses/` grouped per-family.

```python
def expand_method(base_name, entry) -> list[dict]:
    if "__" in base_name:
        raise ValueError(f"Method base name must not contain '__': {base_name!r}")
    template = dict(entry.get("template") or {})
    over = entry.get("over") or {"_dummy": [None]}
    keys = list(over.keys())
    out = []
    for combo in itertools.product(*(over[k] for k in keys)):
        ov = {k: v for k, v in zip(keys, combo) if k != "_dummy"}
        suffix = "".join(f"__{k}={format_over_value(v)}" for k, v in ov.items())
        out.append(method_coordinate(f"{base_name}{suffix}", entry["function"], {**template, **ov}))
    return out

def resolve_method(coord):
    return coord["name"], resolve_callable(coord["function"]), resolve_distributions_in_kwargs(coord["kwargs"])

def run_method(coord, simulation) -> dict:
    name, fn, kwargs = resolve_method(coord)
    return {"method": name, **fn(simulation, **kwargs)}
```

```python
def manifest_dict(library, config) -> dict:
```

```python
import inspect
def _file(fn): return inspect.getfile(getattr(fn, "func", fn))

def simulation_code_files(coord, library):
    spec = resolve_simulation_from_coord(coord)     # build samplers from coord
    fns = [core.simulate, spec.design_sampler, spec.effect_sampler]
    if spec.error_sampler is not None: fns.append(spec.error_sampler)
    return sorted({_file(f) for f in fns})

def method_code_files(mcoord, library): return [_file(resolve_callable(mcoord["function"]))]
def reduction_code_files(reduction, library):
    import importlib; return [_file(getattr(importlib.import_module(f"reductions.{reduction}"), "build"))]
def analysis_code_files(analysis):
    import generate_plots; return [_file(generate_plots.ANALYSIS_RENDERERS[analysis])]
```
