# Logistic SuSiE Experiments

Standalone Snakemake pipeline for comparing logistic SuSiE implementations in
the [`gibss`](https://github.com/karltayeb/gibss-mono) package. Parallels
`twogroup_experiments/` but with a simplified two-axis simulation model and a
focus on the approximation methods rather than the two-group model.

## Methods compared

One method per gibss logistic SuSiE implementation:

- `globaljj` — global Jaakkola–Jordan variational bound
- `localjj` — local JJ bound
- `quadrature` — `logistic_quadrature`, the accurate reference

## Simulation model

Two axes only — **design** × **enrichment** (no signal / error layer):

- `design_sampler(rng) -> X` — gene sets (`hallmark`) or Markov designs.
- `effect_sampler(X, rng) -> (causal_indices, causal_effects)` — the enrichment.
- `y ~ Bernoulli(sigmoid(intercept + X @ b))` — well-specified for logistic SuSiE.

Specs are pure data, content-addressed by hash; realized simulations are
regenerated on demand and fit results are the cache boundary.

## Layout

| Path | Role |
|---|---|
| `core.py` | `SimulationSpec`, `simulate()`, fit-state extraction, hashing |
| `simulations/` | `design/` and `effect/` samplers |
| `fits/logistic.py` | parametrized fit dispatching over the three impls |
| `reductions/` | per-batch `pip` and `cs` summaries |
| `experiments/library.yaml` | designs, enrichments, methods, reductions |
| `experiments/*.yaml` | supercollections (declarative collection grids) |
| `experiments/loader.py` | coordinate/manifest resolution |
| `logistic_susie_experiments.snk` | Snakemake workflow |

## Run

```bash
# default target: simulate -> fit -> reduce for every (batch, method)
uv run snakemake -s logistic_susie_experiments.snk reduce_all -j8

# intermediate targets
uv run snakemake -s logistic_susie_experiments.snk simulate_all -j8
uv run snakemake -s logistic_susie_experiments.snk fit_all -j8
```

Results land under `results/by_batch/{batch_hash}/...`.
